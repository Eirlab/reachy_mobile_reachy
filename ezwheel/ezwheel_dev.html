Reachy Mobile

Ezwheel

Trois principaux blocs ont été développés pour la gestion de la base mobile : le développement du package ros reachy_mobile_ezwheel (contenant les packages reachy_mobile_navigation, reachy_mobile_description , reachy_mobile_slam et reachy_mobile_teleop), le développement d'une API permettant d'exposer les fonctions de contrôle de la base ainsi que de la classe SimpleNavigationGoals permettant de gérer les objectifs de navigation.

SimpleNavigationGoals

Cette classe a pour but de générer les différents messages nécessaire pour communiquer avec le robot en utilisant l'action serveur move_base.

__init__

Cette fonction instancier la classe et récupère un accès au move_base action server. Elle initialise aussi les différents champs des messages qui seront envoyés.

go_to

Crée et envoie un message sur le topic /move_base_simple/goal pour se déplacer sur la carte

Paramètres

  • x: Position en x
  • y: Position en y
  • theta: Orientation
  • wait_for_result: timeout avant d'annuler le déplacement
  • frame: Nom de la frame dans laquelle se trouve le point
  • blocking: Si True, la fonction bloque jusqu'à ce que le robot arrive à la position

is_arrived

Renvoie le GoalStatus de la dernière requête envoyée parmis la liste suivante : "Running", "Rejected", "Preempted" , "Aborted", "Succeeded", "Lost" or "Undefined".

cancel_all_goals

Met fin à tous les déplacements en cours et tous les objectifs en attente.

API

L'API est une api flask nous permettant de transmettre un ordre de navigation, d'obtenir le statut de la navigation et de l'annuler.

goal

  • Méthode : POST
  • Route : /goal
  • Données : {"x": float, "y": float, "theta": float}
  • Résumé : Envoie un ordre de navigation

status

  • Méthode : GET
  • Route : /status
  • Données : Aucune
  • Résumé : Renvoie le statut de la navigation

cancel

  • Méthode : POST
  • Route : /cancel
  • Données : Aucune
  • Résumé : Annule la navigation en cours
